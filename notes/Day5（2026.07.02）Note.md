深入理解以下核心概念，并思考它们在 GeoAI 空间分析和数据清洗中的重要应用：

1. **多分支判定（`elif`）的逻辑截断性**：多分支结构是从上到下依次判定的。一旦某个条件满足（为 `True`），执行完该代码块后会直接跳出整个 `if...elif...else` 结构。因此，**在处理具有包含关系的区间判定时（例如降雨量分级或 NDVI 植被覆盖度分级），必须严格注意条件的先后编写顺序**。
2. **`match-case` 模式匹配的优雅性**：在处理遥感影像类别标签（如 "Water", "Build", "Forest"）或科研任务状态等固定、明确的值时，`match-case` 比冗长的 `if-elif` 代码可读性更高，结构更清晰。

### 实验任务

创建 `weather_advice.py` 文件。手工编写并运行以下代码，前半部分利用 `if...elif...else` 结合你的 **GIS/遥感专业背景**，模拟无人机外场数据采集的气象判定；后半部分结合**考研复试线及导师团队选择**场景，通过 `match...case` 语法实现智能匹配：

Python

```
# -*- coding: utf-8 -*-
"""
程序名称：weather_advice.py
作者：黄乐源
日期：2026-07-02
描述：掌握if-elif-else多分支与match模式匹配，模拟天气预警与科研方向匹配
"""

print("================= 任务 1：if-elif-else 多分支（遥感气象与外场数据采集建议） =================")
# 模拟输入当前的24小时累积降雨量（单位：mm），并根据气象级别给出无人机（UAV）外场作业建议
input_rain_str = input("请输入当前的24小时累积降雨量 (mm): ")
rainfall = float(input_rain_str)

# 严格注意区间从高到低的先后判定顺序（大范围/严格条件在前）
if rainfall >= 50.0:
    warning_level = "暴雨预警"
    advice = "🔴 严禁任何无人机外场数据采集作业，注意防汛防涝！"
elif rainfall >= 25.0:
    warning_level = "大雨"
    advice = "🟡 天气恶劣，设备极易受损，建议取消野外光谱数据采集计划。"
elif rainfall >= 10.0:
    warning_level = "中雨"
    advice = "🔵 降雨明显，空气湿度过大，会干扰红外波段反射率，不建议作业。"
elif rainfall > 0.0:
    warning_level = "小雨"
    advice = "⚪ 局部有微量降水，如需作业请务必做好设备防水。"
else:
    warning_level = "晴朗/无雨"
    advice = "🟢 气象条件良好，非常适合无人机外场遥感影像采集！"

print(f"📊 气象评估结果：当前天气为【{warning_level}】")
print(f"💡 专家建议：{advice}")


print("\n================= 任务 2：Python 3.10+ 全新 match...case 语法（预研团队方向匹配） =================")
# 模拟匹配你即将要调研的华中师范大学资源与环境学院的导师科研方向
print("--- 华师大 GeoAI 研究方向智能匹配系统 ---")
print("可选关注导师简称: Liu (刘鹏程老师) / Zhou (周杨老师)")
mentor = input("请输入你想查询的导师简称: ").strip()

# 使用全新的 match-case 语法替换繁琐的字符串相等判断
match mentor:
    case "Liu" | "liu" | "刘鹏程":
        print(" 👨‍🏫 匹配成功：刘鹏程老师课题组")
        print(" 🎯 核心研究：基于 Transformer 框架的建筑物边缘优化提取研究、遥感智能解译。")
    case "Zhou" | "zhou" | "周杨":
        print(" 👨‍🏫 匹配成功：周杨老师课题组")
        print(" 🎯 核心研究：请关注最新的资源与环境遥感应用研究成果。")
    case _:
        print(" 💡 提示：输入了其他导师，建议开学后在资环学院官网进一步查询。")
```
