#### 实验任务

创建 `multiplication_table.py` 文件。手工编写并运行以下代码，前半部分利用双层 `for` 循环嵌套实现经典的九九乘法表，并将其与 GIS 中的行列扫描对齐；后半部分结合 `while` 循环和 `break` 语法实现极具交互性的“猜数字综合系统”：

Python

```
# -*- coding: utf-8 -*-
"""
程序名称：multiplication_table.py
作者：黄乐源
日期：2026-07-04
描述：掌握for循环、range()序列、双层循环嵌套以及break/continue控制流，实现九九乘法表与猜数字综合案例
"""
import random

print("================= 任务 1：嵌套循环（双层 for 循环输出九九乘法表） =================")
# 模拟 GIS 二维空间格网的行列扫描机制：外层循环控制行（Row），内层循环控制列（Column）
for row in range(1, 10):         # 行控制：从第 1 行到第 9 行
    for col in range(1, row + 1): # 列控制：当前行的最大列数不超过当前行号
        # 使用 f-string 进行格式化对齐，end="\t" 确保同行的乘法公式通过制表符横向排列
        print(f"{col} * {row} = {col * row}", end="\t")
    print()  # 内层“列”循环一轮结束，执行换行，进入外层下一“行”扫描


print("\n================= 任务 2：循环中断控制综合实战（智能猜数字交互系统） =================")
# 融合随机数生成、条件分支、无限循环与 break 强行中断机制
target_number = random.randint(1, 100)
guess_count = 0

print("🎮 --- 欢迎来到华师大资环学院 Python 基本功测试：猜数字系统 ---")
print("💡 系统已经随机生成了一个 1 到 100 之间的整数，请开始你的预测。")

while True:
    input_str = input("请输入你猜的数字 (1-100) [输入 q 退出]: ").strip()
    
    # 支持用户中途主动退出
    if input_str.lower() == 'q':
        print(f"👋 退出游戏。很遗憾你未能完成测试，正确答案其实是：{target_number}")
        break
        
    guess_num = int(input_str)
    guess_count += 1
    
    # 条件多分支判定
    if guess_num > target_number:
        print("📈 提示：猜大啦！目标数字比这要小哦。")
    elif guess_num < target_number:
        print("📉 提示：猜小啦！目标数字比这要大哦。")
    else:
        print(f"🎉 恭喜你！完全匹配正确！正确答案就是【{target_number}】")
        print(f"📊 统计：你总共尝试了 {guess_count} 次。基本功扎实，算法闭环达成！")
        # 匹配成功，使用 break 强行跳出当前 while 无限循环，结束程序
        break

print("🏁 任务结束：恭喜攻克循环结构全部核心知识点！")
```

## 今日收获

**30 for循环：**

for循环本质是一种**轮询遍历**机制，对一批内容进行逐个处理。

**31 range案例：**

range语句用于生成指定规则的数字序列。主要用法有三种：

1、**range(end)**，即获取一个**从0开始到end结束**的数字序列，但**不含end本身**。

2、**range(start,end)**，即获取一个**从start开始到end结束**的数字序列，但**不含end本身**。

3、**range(start,end,step)**，即获取一个**从start开始到end结束**的数字序列，但**不含end本身**，**step为步长**（即每两个数字间的间隔）

## 遇到的问题/注意事项

**for循环**与**while循环**的**对比**：

**while循环**用于**在某个条件满足时一直循环**，循环次数通常**未知**，只知道循环开始/结束的条件。

**for循环**用于**对一个已知的数据集进行遍历**或**已知次数的循环**。（关注的是遍历每一个元素）

注意：range数字序列**不包含end本身**。所以在输出99乘法表时为了range能包含外层循环的数字，需要将range中的end值+1，如：

```python
for a in range(1,10): 
    for b in range(1,a+1):
```

在嵌套循环时，如果需要输出循环的元素值，需要格外注意range的起始值。

在上传文件至Github仓库时，如果注释写错了并且已经上传了，可以按照下面的方法修正：

```

```

运行完这行命令后，你在**本地**的错误备注就已经被新备注完美覆盖了。

**第二步：将正确的备注强推到 GitHub 仓库**

由于你之前已经把错误的日志推上去了，此时本地和远程的提交历史会出现冲突。我们需要使用 `force`（强推）命令，用本地刚刚修改好的正确历史去**覆盖** GitHub 远程的历史：

```

```

> ⚠️ **学术规范提醒**：`--force`（或 `-f`）是一个威力很大的命令，它会无条件覆盖远程分支。但在**你个人的独立开发仓库**（比如你的 `GeoAI-Learning` 个人预研仓库）中，这是修正自己提交历史的**标准且唯一的手段**。开学后如果进入多人协作的团队大项目，切记先和组员沟通好再使用。
