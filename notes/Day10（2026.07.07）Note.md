### 第二阶段：理论学习（约 0.5 小时）

深入理解以下核心概念在 GeoAI 空间分析与遥感多源数据处理中的重要桥梁作用：

1. **集合（Set）在多维遥感标签重叠区剔除与多源分类一致性比对中的应用**：

   在做监督分类（Supervised Classification）时，从不同的遥感源（如 Sentinel-2、Landsat-9 或高分影像）中获取的训练样本标签，往往含有大量重合、冗余的非结构化地物类别命名。通过集合的天然去重机制 `set()`，能一瞬间洗掉重复项，保留唯一的“核心分类体系”。此外，利用集合的差集（`difference`）与交集（`intersection`），可以以毫秒级速度审计出两个遥感解译分类器在结果分类标签上的一致性或冲突部分。

2. **字典（Dict）作为多属性空间要素（GeoJSON）与模型超参数（Hyperparameters）的高效映射仓库**：

   字典是空间矢量要素（Feature）属性表的核心映射基础。在标准的地理数据传输格式 GeoJSON 中，属性信息都是以 `{"properties": {"class": "building", "area": 1250, "confidence": 0.94}}` 的形式组织。不仅如此，在 PyTorch 深度学习阶段，模型的超参数（如 `{"learning_rate": 0.001, "batch_size": 32, "backbone": "ResNet-50"}`）也是通过字典来无缝管理、传递和保存的。掌握了字典，就拿到了高效数据调度的金钥匙。

#### 实验任务

创建 `word_frequency.py` 文件。手工编写并运行以下代码，前半部分掌握集合与字典的基础高级操作，后半部分构建一个模拟“多源遥感影像地物分类标签处理与多特征要素属性精准分类检索系统”：

```
"""
程序名称：word_frequency.py
作者：黄乐源
日期：2026-07-07
描述：攻克集合高效去重与交并差运算，掌握字典的键值映射与嵌套提取，模拟多源地物标签分析
"""

print("================= 任务 1：集合（Set）的去重与空间拓扑关系运算 =================")
# 模拟从两个不同分类模型中提取出的遥感地物覆盖分类标签列表（存在大量重复与杂质）
model_a_labels = ["building", "water", "building", "vegetation", "road", "barren"]
model_b_labels = ["vegetation", "road", "vegetation", "cropland", "shadow", "water"]

print(f"模型 A 提取的原始标签: {model_a_labels}")
print(f"模型 B 提取的原始标签: {model_b_labels}")

# 1. 利用集合天然去重的特性
set_a = set(model_a_labels)
set_b = set(model_b_labels)
print(f"1. 模型 A 唯一地物集合: {set_a}")
print(f"2. 模型 B 唯一地物集合: {set_b}")

# 2. 集合运算：并集（求两个模型一共捕获了多少种地物类别）
all_classes = set_a.union(set_b)
print(f"3. 联合地物类别（并集）: {all_classes}")

# 3. 集合运算：交集（求两个模型达成共识的地物类别）
common_classes = set_a.intersection(set_b)
print(f"4. 共识地物类别（交集）: {common_classes}")

# 4. 集合运算：差集（求模型 A 独有而模型 B 没有检测到的地物类别）
unique_a = set_a.difference(set_b)
print(f"5. 模型 A 特有类别（差集）: {unique_a}")


print("\n================= 任务 2：字典（Dict）键值映射与要素检索 =================")
# 模拟一个典型的 GIS 矢量建筑物要素（GeoJSON Properties）属性映射
building_feature = {
    "feature_id": 1024,
    "class_name": "building",
    "area_m2": 458.5,
    "height_m": 18.5,
    "coords_wgs84": (114.3721, 30.5312)  # 使用 Day09 学过的元组锁死坐标
}
print(f"原始建筑物要素字典: {building_feature}")

# 1. 字典的访问 (Key 访问)
print(f"要素 ID: {building_feature['feature_id']}, 地物类别: {building_feature['class_name']}")

# 2. 字典的修改与添加新键值对
building_feature["floors"] = 6            # 动态添加层数属性
building_feature["area_m2"] = 460.0        # 修改面积精度
print(f"更新后的要素字典: {building_feature}")

# 3. 字典的成员判断与安全获取（防止因 Key 不存在导致报错崩溃）
# 推荐在处理脏数据时使用 get() 方法
sensor_type = building_feature.get("sensor", "未知传感器")
print(f"安全属性查询 -> 传感器源: {sensor_type}")


print("\n================= 任务 3：多案例融合终极实战（多属性嵌套数据综合审计系统） =================")
# 模拟华中师范大学城市与环境科学学院 GeoAI 课题组对不同遥感地物斑块的高级统计结构
geo_database = {
    "BLOCK_01": {"class": "building", "area": 1250.2, "confidence": 0.942, "is_active": True},
    "BLOCK_02": {"class": "water", "area": 8450.0, "confidence": 0.985, "is_active": True},
    "BLOCK_03": {"class": "building", "area": 310.4, "confidence": 0.721, "is_active": False},
    "BLOCK_04": {"class": "vegetation", "area": 15600.8, "confidence": 0.891, "is_active": True}
}

print("📊 待分析的多源嵌套地物特征属性数据库：")
for block_id, info in geo_database.items():
    print(f"斑块: {block_id} -> {info}")

print("\n--- 🔎 筛选出置信度(confidence) > 0.85 且处于激活状态(is_active) 的建筑物或水体斑块 ---")

for block_id, info in geo_database.items():
    # 提取多层嵌套的字典属性
    lbl_class = info["class"]
    lbl_conf = info["confidence"]
    lbl_active = info["is_active"]
    
    # 逻辑判断与复合数据帅选
    if lbl_conf > 0.85 and lbl_active and (lbl_class in ["building", "water"]):
        print(f"🎯 筛选通过 -> 斑块编号: {block_id:<10} | 地物类别: {lbl_class:<10} | 置信度: {lbl_conf:.3f}")

print("\n🏁 Day10 集合去重与字典映射安全系统成功建立，准备进行版本控制！")
```

## 今日收获

**50 集合基础**：

集合（set）是一种**无序**（**不支持下标索引访问**）的、**不可重复**、**可修改**的数据容器。集合会**自动去重**，存储**不重复（唯一）**的元素。

**空集合的输出**就是**set()**。

集合中的**常见方法**：

**add(..)**：**添加元素**到集合中；

**remove(..)**：**移除**集合中指定元素（指定元素不存在则报错）；

**pop()**：**随机删除**集合中的元素并返回；

**clear()**：**清空**集合；

**difference()**：求取两个集合的**差集**（包含在集合A但不包含在集合B的元素）；

**union()**：求取两个集合的**并集**；

**intersection()**：求取两个集合的**交集**。

**51 集合案例**：

**&**（与）运算符可用于**集合求交集**，其语法为：`fa_set2 = french_set & art_set`。

**-**（减号）运算符可用于**集合求差集**，其语法为：`fnb_set2 = football_set - basketball_set`。

**集合推导式**用于快速构建集合，语法：**{要往集合中添加的元素 for s in set1 if 条件}**

**|**运算符可用于**集合求并集**，其语法为：`all_set = football_set | basketball_set | french_set | art_set`。

**52 字典介绍**：

**字典（dict）**里面存储的是**键值对（key:value）**类型的数据，每个键都对应一个值，可以**根据键（key）找到对应的值（value）**。

在其他编程语言中，有的叫**映射（map）**，有的叫**哈希表（hash）**，其实质就是字典。其**特点**如下：

**键值对（key:value）存储**、**键不能重复**、**可修改**。

当key重复时，**后面的key**对应的值会**覆盖前面**的值。

**定义空字典**可以直接用**{ }**，其表示的就是空字典，也可以用**dict()**。

## 遇到的问题/注意事项

注意：空集合的定义不可以使用{ }，**{ }**表示的是**空字典**。

注意：字典中的**value可以是任何类型的数据**，而**key不能为可变类型**（如：列表list、集合set、字典dict），换言之	（如：int、float、str、tuple(元组不可修改，故也是不可变类型)）。

注意：`del 字典名称[key]`这种字典删除方法获取不到对应值，但`字典名称.pop(key)`可以。
